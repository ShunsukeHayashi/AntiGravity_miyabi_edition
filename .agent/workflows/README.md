# Workflows - ワークフロー定義

このディレクトリには、一般的な開発タスクのワークフロー定義が含まれています。

## 📋 利用可能なワークフロー

### 1. generate-unit-tests.md
ユニットテストの自動生成ワークフロー

**用途:** 既存のコードに対してテストを追加する

**手順:**
1. テスト対象ファイルを特定
2. テストケースを設計
3. Jestテストコードを生成
4. カバレッジ80%を目標

### 2. implement-feature.md
新機能実装の標準ワークフロー

**用途:** 新しい機能を実装する

**手順:**
1. 要件分析
2. 設計（型定義、インターフェース）
3. 実装（コード生成）
4. テスト作成
5. ドキュメント作成

### 3. refactor-code.md
リファクタリングの標準ワークフロー

**用途:** 既存コードの品質向上

**手順:**
1. リファクタリング対象の特定
2. テストの準備（回帰テスト）
3. リファクタリング実行
4. テスト実行（機能保証）
5. レビュー

### 4. fix-bug.md
バグ修正の標準ワークフロー

**用途:** バグを修正する

**手順:**
1. バグの再現
2. 原因特定
3. 修正実装
4. 回帰テスト作成
5. 動作確認

## 🎯 ワークフロー使用方法

### Claude Code での使用

ワークフローを参照して作業を進める場合：

```
このファイルの内容に従って {タスク} を実行してください：
.agent/workflows/{workflow-name}.md
```

### 新しいワークフローの追加

新しいワークフローを追加する場合は、以下のテンプレートを使用：

```markdown
# {ワークフロー名}

## 概要
{ワークフローの説明}

## 前提条件
- {前提条件1}
- {前提条件2}

## ステップ

### 1. {ステップ名}
{ステップの詳細}

### 2. {ステップ名}
{ステップの詳細}

## 成功基準
- [ ] {基準1}
- [ ] {基準2}

## トラブルシューティング
{よくある問題と解決策}
```

## 📊 ワークフロー一覧

| ワークフロー | 用途 | 所要時間目安 |
|------------|------|------------|
| generate-unit-tests | テスト生成 | 1-2時間 |
| implement-feature | 機能実装 | 4時間-3日 |
| refactor-code | リファクタリング | 2-8時間 |
| fix-bug | バグ修正 | 1-4時間 |
